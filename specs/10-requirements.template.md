# Requirements – TindahanPro

## Functional Requirements

1. The system must allow users to add, edit, and delete products.
2. The system must store product details including name, category, buying price, selling price, and stock.
3. The system must allow recording of sales transactions.
4. The system must automatically compute total sale amount.
5. The system must automatically deduct product stock after each sale.
6. The system must compute profit based on buying and selling price.
7. The system must display daily sales summary.
8. The system must detect and display low-stock items.
9. The system must allow adding and managing suppliers.
10. The system must allow assigning products to suppliers.
11. The system must track customer credit (utang).
12. The system must allow marking credits as paid or unpaid.
13. The system must generate reports (sales, profit, inventory, credit).

## Non-Functional Requirements

1. The system must be user-friendly and easy to navigate.
2. The system must respond quickly to user inputs.
3. The system must maintain data accuracy.
4. The system must be accessible via web browser.
5. The system must ensure data persistence using a database.
